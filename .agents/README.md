# 代理协作工作区

`.agents` 用于承载代理与开发者协作过程中的草案材料，包括需求澄清、技术方案、测试用例、设计决策和待办想法。

这里的内容可以是不完整的、待确认的、未实现的。它用于帮助自然语言讨论逐步收敛为可实现方案。

## 使用规则

- `.agents/docs/` 中的内容默认不是项目事实。
- 已确认并完成实现的内容，再同步或提升到 `docs/`。
- 如果 `.agents/docs/` 与代码或 `docs/` 冲突，默认以代码和 `docs/` 为准。
- 代理可以在这里记录方案推演、备选方案、待确认问题和测试设计。
- 不在这里保存密钥、账号密码、机器本地路径或私人信息。
- 每一次执行产生代码、配置、测试或文档变更时，都要同步更新项目进展和变更记录。
- 同步更新至少包括 `docs/project/status.md` 和 `docs/project/changelog.md`；涉及架构或决策时，还要更新 `docs/project/decision-log.md`。
- 如果变更推进了 `.agents/docs/` 中的需求、方案或测试草案，也要在对应草案中更新“当前实现状态”或“后续待办”。

## 目录说明

- `docs/requirements/`：需求澄清和需求规格草案。
- `docs/designs/`：技术方案、领域建模、接口设计和实现方案草案。
- `docs/test-cases/`：测试用例设计草案。
- `docs/decisions/`：设计决策记录，包含待确认和已确认但尚未提升到 `docs/` 的内容。
- `docs/backlog/`：后续想法、待办事项和未排期功能。
- `templates/`：需求、方案、测试用例和决策记录模板。
