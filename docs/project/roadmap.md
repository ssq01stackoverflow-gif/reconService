# 项目路线图

本文档记录项目阶段目标和演进方向。路线图可以随着需求澄清和实现进展调整。

## 阶段一：基础框架

目标：建立可持续演进的 Spring Boot 分层单体工程。

状态：已完成主体工作。

范围：

- Java 8、Spring Boot 2.7.x、Maven 项目骨架。
- 基础健康检查接口。
- 统一响应与全局异常处理。
- DDD 分层包结构。
- 代理协作草稿区和项目事实文档区。
- 项目状态、路线图、变更记录和决策索引。

## 阶段二：核心领域模型

目标：定义核对系统的最小可用领域模型。

范围：

- `ReconTask`：核对任务定义。
- `ReconBatch`：一次任务执行批次。
- `ReconRule`：核对规则或规则组。
- `ReconSourceRecord`：源侧数据快照。
- `ReconTargetRecord`：目标侧数据快照。
- `ReconDiffResult`：核对差异结果。
- `ReconStatus`：任务、批次或结果状态。

## 阶段三：任务管理

目标：支持创建、查询和维护核对任务定义。

范围：

- 任务创建。
- 任务查询。
- 任务启停或状态维护。
- 任务规则配置的第一版表达方式。
- Controller、Application Service、Domain、Repository 的端到端骨架。

## 阶段四：核对执行

目标：支持按任务发起批次，并执行确定性的核对匹配与比较。

范围：

- 批次创建。
- 源侧和目标侧快照接入。
- 按核对键匹配记录。
- 字段、金额、状态、时间和派生值比较。
- 差异结果生成。
- 核对逻辑单元测试。

## 阶段五：结果查询与人工处理

目标：支持查询差异结果，并区分自动核对结果与人工处理动作。

范围：

- 批次结果查询。
- 差异明细查询。
- 人工确认、忽略或备注。
- 差异可追溯到任务、批次、源记录、目标记录和规则。

## 阶段六：准实时接入与运维能力

目标：支持准实时数据接入和基础运维检查。

范围：

- 消息或外部系统接入适配。
- 数据接入幂等。
- 手动重试。
- 定时或异步执行入口。
- 运维检查接口。
- 关键指标和日志。
