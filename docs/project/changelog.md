# 项目变更记录

本文档记录对开发者、维护者和后续代理有意义的项目级变更。它不是 commit 日志的替代品，而是人类可读的变更摘要。

## 记录规则

- 记录架构、领域、接口、行为、测试和文档治理方面的重要变化。
- 不记录纯格式化、无语义变化或临时草稿调整。
- 每条记录应说明变化内容和影响。
- 代码层面的完整细节仍以 Git 历史为准。
